## 1. 视觉尺寸的实际实现

高度、宽度、整体大小排序使用页面实际显示图片中的非透明区域，而不是整张 PNG，也不是人工厘米。

实现函数：`displayedOpaqueMetrics(item)`

```js
显示非透明宽 = displayWidthPx * contentWidthPx / widthPx
显示非透明高 = displayHeightPx * contentHeightPx / heightPx
显示非透明面积 = 显示非透明宽 * 显示非透明高
```

进入排序字段后：

- `sort.height`：显示非透明高。
- `sort.width`：显示非透明宽。
- `sort.size`：显示非透明面积。

视觉排序的 6 个序列点通过 `spreadSequenceItems` 从排序结果中拉开取样，目前取点是：

```text
0%, 30%, 58%, 78%, 92%, 100%
```

这样示例和答案会比普通均分更明显。

## 2. 三类关卡的通用实现

### 排序题

例如高度、宽度、大小、颜色。

流程：

1. 调用规则的 `pool()` 得到候选物体池。
2. 用规则的 `compare(item)` 排序。
3. 从排序后结果中取 6 个物体。
4. 前 3 个是示例，后 3 个是玩家答案。
5. 判题时按 `answerIds[index]` 逐格检查顺序。

### 流程题

例如晨间护肤、洗护护理、快速出门妆。

流程：

1. 用 `onePerStage(...)` 或 `onePerProductOption(...)` 生成固定流程的 6 个物体。
2. 前 3 个是示例，后 3 个是答案。
3. 玩家池会加入 3 个干扰项。
4. 判题时按 `answerIds[index]` 逐格检查顺序。

### 找同类题

例如找粉紫色、找圆罐、找洁面类。

流程：

1. `buildExcludePool(...)` 或专用 pool 生成正确物体池与干扰池。
2. 正确池前 3 个作为示例。
3. 正确池第 4-6 个作为答案集合。
4. 干扰池取最多 12 个。
5. 判题只检查玩家放入的 3 个是否等于答案集合，不检查摆放顺序。

注意：这里是“集合判题”，不是顺序判题。

## 3. 30关主线规则

### 第1关：高度从低到高

- 类型：排序题。
- 物体池：所有产品。
- 排序字段：`sort.height`。
- 依据：页面显示非透明高度。
- 干扰项：无。

### 第2关：晨间护肤流程

- 类型：流程题。
- 阶段：清洁 → 水类 → 精华 → 保湿 → 保湿 → 防晒。
- 物体池：每个阶段选一个。
- 干扰项：3个。

### 第3关：找出粉紫色物品

- 类型：找同类题。
- 正确条件：`colorFamily` 是粉红或紫色。
- 判题：只检查答案集合，不检查顺序。

### 第4关：横向宽度从窄到宽

- 类型：排序题。
- 物体池：所有产品。
- 排序字段：`sort.width`。
- 依据：页面显示非透明宽度。

### 第5关：夜间护肤流程

- 类型：流程题。
- 阶段：卸妆清洁 → 清洁 → 水类 → 精华 → 眼部护理 → 保湿。
- 干扰项：3个。

### 第6关：找出圆罐

- 类型：找同类题。
- 正确条件：`shapeType === "圆罐"`，并排除眼霜。
- 判题：只检查答案集合，不检查顺序。

### 第7关：高度从高到低

- 物体池：所有产品。
- 排序字段：`-sort.height`。

- 典型序列：洗发水 → 护发素 → 发膜 → 香皂 → 身体乳 → 护手霜。
- 干扰项：3个。
